# A733 XWayland 24.1.6 Glamor 测试

## 目标

验证更新版 XWayland 能否在 KWin 使用闭源 PowerVR renderer、`card0` 继续负责
HDMI KMS 输出的条件下，解决 glamor 软件回退。

## 构建

在 A7Z 上原生构建了与 Debian 13/Trixie 对齐的 XWayland `24.1.6`，构建环境为
Debian 12 加 Bookworm backports：

- XWayland 源码 SHA-256：
  `737e612ca36bbdf415a911644eb7592cf9389846847b47fa46dc705bd754d2d7`；
- 本地前缀安装 xorgproto `2024.1` 头文件；
- libdrm `2.4.123`、Mesa/GBM `25.0.7`、wayland-protocols `1.43`；
- 单线程构建，临时增加 2 GiB swap；
- 最终 ARM64 二进制 SHA-256：
  `a58c2908da43d45bdf28f63b692c260452b9f455ce9aa36cd52a33190f389135`。

二进制只安装在 `/home/radxa/xwayland-build/prefix`，没有替换 Debian 的
`xwayland 22.1.9-1` 软件包。

## 结果

| XWayland | 图形环境 | 结果 |
|---|---|---|
| `22.1.9` | 发行版 Mesa | `egl: failed to create dri2 screen`，回退软件绘制 |
| `22.1.9` | PowerVR DRI/EGL | 加载 `pvr_dri.so` 后仍回退软件绘制 |
| `24.1.6` | 发行版 Mesa | `EGL setup failed, disabling glamor` |
| `24.1.6` | PowerVR DRI/EGL | `Main linux-dmabuf device has no render node` |

所有组合中，XWayland 都同时打开 `/dev/dri/card0` 和
`/dev/dri/renderD128`，因此问题不是缺少文件描述符或访问权限。

## 多 GPU 设备顺序测试

还测试了 `KWIN_DRM_DEVICES=/dev/dri/card1:/dev/dri/card0`，尝试把 PowerVR
节点作为 KWin 主 DRM 设备，但 `card1` 不是完整的 KMS 设备，结果为：

- `card1` 不支持 KWin 所需的 atomic modesetting；
- KWin 无法创建 linux-dmabuf feedback table；
- 跨 GPU AR24 framebuffer 创建返回 `EINVAL`；
- XWayland 退出，Plasma 启动超时。

该配置已经通过 SSH 立即恢复。

## 结论

仅升级 XWayland 无法启用 glamor。当前拓扑是：KMS 显示设备 `card0` 没有
render node，而闭源渲染设备 `card1`/`renderD128` 又不具备 KWin 对主 DRM GPU
要求的 KMS 能力。下一步应在保持 `card0` 为主 KMS 设备的前提下，研究或修改
KWin 针对这种分离拓扑的 linux-dmabuf feedback 选择逻辑。

板卡已恢复为 XWayland `22.1.9`、`gpu6` 清洁 wrapper、`default l0` 和正常
SDDM 登录界面；临时自动登录及 swap 文件均已删除。

## Render Node Feedback 后续验证

2026-07-17 临时将 KWin 的 linux-dmabuf feedback 主设备改为 EGL render node
`/dev/dri/renderD128`，同时继续使用 `/dev/dri/card0` 负责 KMS。XWayland
`24.1.6` 随后越过此前的 `Main linux-dmabuf device has no render node` 错误，
并成功加载 `pvr_dri.so`。这验证了 KWin render node 选择补丁的方向。

XWayland 硬件加速仍未完成，下一层问题位于 PowerVR EGL/glamor 路径：

- 桌面 OpenGL context 建立时报 `Failed to make EGL context current with GL`；
- glamor 检测到 depth 30 read format/type 不符合预期并回退；
- DRI3 无法创建 PVR screen；
- `glxinfo -B` 显示 Mesa llvmpipe，`Accelerated: no`。

因此剩余工作已经从 linux-dmabuf 设备发现推进到 XWayland 与闭源 PowerVR
EGL/DRI 栈的兼容性。测试完成后已删除临时 override、自动登录和 swap，系统
wrapper 恢复为 `gpu6` 配置与 Debian XWayland `22.1.9`。
