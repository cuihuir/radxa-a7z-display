# 贡献指南

## 目标

让这个仓库便于维护，适合长期开展 A733 显示相关工作。

## 规则

- 每次修改尽量保持小而聚焦。
- 先更新英文文档，再更新对应的中文翻译。
- 影响项目方向的重要技术决策要记录到决策日志。
- 只要硬件、BSP 或发布信息发生变化，就要补充或更新来源。
- 文件名和目录位置要保持清晰、可预测。
- 流程保持轻量。如果只有自己维护，就把这些规则当成自我约束，不要搞成形式主义。

## 工作流

1. 打开或更新对应的文档对。
2. 如果修改影响项目方向，把变更记录到决策日志。
3. 如果使用了新的参考资料，更新来源索引。
4. 检查链接、措辞和一致性。
5. 使用有描述性的提交信息完成提交。

## 评审要求

- 中英双语术语必须保持一致。
- 不要在多个文件里重复定义同一件事。
- 如果结论不确定，要明确标注不确定，而不是写成确定事实。
