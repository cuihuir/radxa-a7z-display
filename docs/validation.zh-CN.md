# 验证指南

## 目的

定义如何证明一条结论、一次构建或一个启动步骤是真实可复现的。

## 验证层级

1. 文档验证
   - 检查链接是否可达。
   - 检查英文和中文版本的范围是否一致。
   - 检查所有结论是否有来源支撑。

2. 构建验证
   - 记录完整的构建命令。
   - 记录板卡、镜像类型和发行版本。
   - 记录成功或失败，并附上关键日志片段。

3. 启动验证
   - 记录使用的显示路径，例如 Micro HDMI 或 USB-C DP Alt Mode。
   - 记录桌面会话是否成功启动。
   - 记录图形加速是否可用，还是只能使用软件渲染。

## 硬件结论的最低证据

- 板卡名称和版本。
- 内核或镜像版本。
- 使用的显示路径。
- 实际观察结果。
- 结果来源，例如日志、截图或可复现的命令输出。
