# 显示栈架构

## 目标

定义在 A733 开发板上点亮并长期维持 HDMI 桌面所需的最小显示栈。

## 分层

1. 引导器和固件
   - 选择正确的板级配置。
   - 加载正确的设备树和内核参数。

2. 内核与 DRM/KMS
   - 提供面板、HDMI 和接口识别。
   - 处理 framebuffer 交接和模式设置。

3. 用户态图形栈
   - 启动显示管理器或合成器。
   - 加载所需的 GPU 用户态组件。

4. 桌面会话
   - 启动一个足够轻量、可用于验证显示、输入和基础渲染的桌面环境。

## 设计原则

- 优先选择最简单且稳定的硬件路径。
- 除非证据表明另一条路径同样稳定，否则将 Micro HDMI 作为主启动路径。
- 将厂商特有组件隔离出来，便于更新或替换，而不是重写整套系统。
- 对每一项硬件相关假设都明确写下来。

## 风险

- GPU 用户态兼容性可能落后于内核支持。
- USB-C 显示路径可能依赖不同且不够稳定的子系统。
- Debian 版本升级可能让某一层失效，即便另一层仍能启动。
