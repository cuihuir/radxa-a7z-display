# Naming Conventions

## Repository name

Use `radxa-a7z-display` for the repository and project identity.

## File naming

- English documents use `kebab-case` names.
- Chinese translations use the same base name with `.zh-CN.md`.
- Keep paired documents in the same directory.

## Directory naming

- Use functional directories such as `docs/research` and `docs/architecture`.
- Do not create duplicate topical directories in multiple languages.

## Decision log entries

- Use short, concrete titles.
- Include the date in the file name if the entry is meant to be a standalone record.
- Prefer one decision per entry when possible.
