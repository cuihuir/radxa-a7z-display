# A733 触摸与异常断电恢复完整镜像

`v0.3.3-a733-touchscreen-recovery` 是面向 Radxa Cubie A7Z/A733 的新版可直接
烧录 Debian 12 KDE + PowerVR 镜像。它在一个版本中整合 `display4`/`gpu8`
热插拔更新、打包版 XWayland/KWin 加速、串口诊断和已验证的挂载前 ext4
恢复路径。

最终的 `display4` 重构通过 `CONFIG_HID_MULTITOUCH=y` 恢复 USB HID 多点触摸。
该内核已在目标 A7Z 上实际启动，并确认连接的触摸屏工作正常后才重新打包镜像。

## 恢复设计

镜像保留三条相互独立的路径：

- `l0`：正常的 `5.15.147-21.1-a733` PowerVR 加速桌面；
- `l1`：屏蔽 `pvrsrvkm` 的原厂内核回退项；
- `a7z-recovery`：保留详细输出的 14.7 MB initramfs，在挂载
  `/dev/mmcblk0p3` 前执行 `fsck.ext4 -a`。

默认项仍为 `l0`。U-Boot 菜单应通过 UART0、`115200 8N1` 查看和选择；恢复时
不能依赖 HDMI 显示文本菜单。

## 实机验证

2026 年 7 月 21-22 日使用独立的 58.2 GiB SD 卡完成验证：

1. 克隆卡通过正常 `l0` 启动，Wi-Fi 和 Plasma 正常；
2. 离线使用 `debugfs` 注入 ext4 块组描述符校验和及空闲块计数错误；
3. 正常 `l0` 无法挂载根分区并进入 initramfs；
4. 通过 UART 选择 `a7z-recovery`；
5. early `fsck.ext4 -a` 将两项错误报告为 `FIXED`，返回状态 `1`；
6. 系统继续进入登录提示，Wi-Fi 恢复，根文件系统状态为 `clean`。

正常 initramfs 还移除了不会在 A733 上使用的 AMD/NVIDIA PCI GPU 模块和固件，
从不稳定的 42.25 MB 边界降到约 19.8 MB，同时不影响 A733 显示、PowerVR、
存储或 Wi-Fi 路径。修改后连续两次正常重启均通过。

导出镜像不包含 NetworkManager 配置、machine identity、SSH host key、journal
或用户历史。首次启动时 systemd 会生成新的 machine ID，
`a7z-ssh-host-keys.service` 会在 `ssh.service` 前生成唯一的 SSH host key。
最终 12 GiB 紧凑镜像已重新写回备用卡，并使用 `radxa/radxa` 到达串口登录提示。

## 烧录

```bash
xz -d radxa-a733-debian12-kde-pvr-recovery-20260722.img.xz
sudo dd if=radxa-a733-debian12-kde-pvr-recovery-20260722.img \
  of=/dev/<目标磁盘> bs=4M status=progress conv=fsync
sync
```

目标卡容量必须不小于解压后的镜像。烧录会清除目标设备上的全部数据，务必确认设备名。

## 已知限制

- 正常根分区无法挂载时，目前需要串口选择恢复菜单；HDMI 不能可靠显示 U-Boot 文本菜单。
- `e2fsck -a` 只自动修复被认为安全的错误；更严重的损坏仍需离线执行
  `e2fsck -fy` 并依赖备份。
- 桌面 GLX 仍为软件渲染；XWayland EGL/GLES 和 KWin 使用打包的 PowerVR 路径。
