# A733 HDMI Hotplug And Boot Recovery Update

This patch release updates the verified Debian 12 KDE + PowerVR system from
`display2`/`gpu7` to `display3`/`gpu8`.

## Fixes

- HDMI unplug/replug now recovers automatically without a udev rule or manual
  xrandr cycle.
- The HDMI HPD handler no longer disables hardware outside DRM atomic state.
- The kernel package explicitly includes the A7Z DTB required by `l0`.
- Kernel and GPU deployment tools restore the independent vendor DTB and
  `module_blacklist=pvrsrvkm` on recovery entry `l1` after `u-boot-update`.
- `gpu8` accepts the compatible `display2` through `display3` kernel package
  range instead of pinning one exact package revision.

## Install

Keep recovery entry `l1` available and install the kernel before the GPU
package:

```bash
sudo tools/deploy_a733_display_kernel.sh \
  linux-image-5.15.147-21.1-a733_5.15.147-21.1+display3_arm64.deb \
  --activate
sudo tools/deploy_a733_gpu.sh \
  a733-pvr-gpu_24.2.6603887+gpu8_arm64.deb \
  --activate
sudo reboot
```

## Verified Result

Hardware validation on July 20, 2026 passed:

- `l0` boots kernel `5.15.147-21.1-a733` from package
  `5.15.147-21.1+display3`;
- `pvrsrvkm 24.2.6603887` loads and creates `card1`/`renderD128`;
- SDDM selects native `1024x600@60Hz` on HDMI-A-1;
- physical HDMI disconnect/reconnect restores the display automatically;
- AIC8800 Wi-Fi, SSH, DKMS modules, and recovery entry `l1` remain available.

This release contains update packages, deployment tools, release notes, and
SHA-256 checksums. It does not replace the full image published with `v0.3.0`.
