# v0.2.0 A733 EDID Display

Date: 2026-07-14

## Scope

This release provides the verified HDMI native-mode fix for the Radxa Cubie
A7Z. It is a display-policy release, not a replacement Debian image.

## Verified Hardware Result

- Board: Radxa Cubie A7Z.
- Base kernel: vendor `5.15.147-21-a733` (`l1`).
- Panel: `FLY-HDMI-LCD7`.
- Selected mode: `1024x600@60Hz`.
- Result: complete KDE desktop, without stretching or cropping.

## Assets

- `0001-drm-prefer-edid-native-mode.patch`: reproducible BSP source patch.
- `sun60i-a733-cubie-a7z.dtb`: prebuilt A7Z DTB for the vendor `l1` kernel.
- `SHA256SUMS`: asset checksums.

## Installation

Back up the vendor DTB, then install the release DTB. This does not replace
the kernel, initramfs, or Wi-Fi modules.

```bash
sudo cp -an /usr/lib/linux-image-5.15.147-21-a733/allwinner/sun60i-a733-cubie-a7z.dtb \
  /usr/lib/linux-image-5.15.147-21-a733/allwinner/sun60i-a733-cubie-a7z.dtb.pre-v0.2.0
sudo install -m 0644 sun60i-a733-cubie-a7z.dtb \
  /usr/lib/linux-image-5.15.147-21-a733/allwinner/sun60i-a733-cubie-a7z.dtb
sync
sudo reboot
```

Verify with:

```bash
sudo journalctl -b -k --no-pager | grep -E 'Configuration mode|drm hdmi mode set'
```

Expected: `1024x600@60Hz` and `drm hdmi mode set: 1024*600` on the tested
panel.
