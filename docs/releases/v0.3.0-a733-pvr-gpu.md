# A733 PowerVR GPU First Port

This release ports the vendor PowerVR BXM DDK to the verified A7Z
`5.15.147-21.1-a733` display kernel:

- `pvrsrvkm.ko` with DDK `24.2@6603887` and BVNC `36.56.104.183`;
- firmware `rgx.fw.36.56.104.183`;
- matched EGL/GLES/Vulkan/OpenCL/DRI userspace under
  `/opt/a733-pvr/24.2.6603887`;
- no vendor `Xorg`, `modesetting_drv.so`, `libglamoregl.so`, or LightDM files.

The package pins the existing system Xorg modesetting driver to display KMS
device `/dev/dri/card0`; the new PowerVR `/dev/dri/card1` and `renderD128`
remain dedicated to acceleration.

Debian marks `/usr/bin/kwin_wayland` with `cap_sys_resource`, which makes the
dynamic loader ignore `LD_LIBRARY_PATH`. Activation therefore creates an
unprivileged, same-version shadow copy under `/usr/local/lib/a733-pvr` and pins
KWin's DRM display backend to `card0`. Disabling the package removes the copy.

Offline validation proves the expected vermagic, DDK and BVNC identifiers, and
that all 285 module imports exist in the current kernel's `Module.symvers`.
A7Z hardware validation also passes module and firmware startup, render-node
creation, Vulkan, OpenCL, EGL/GBM, and accelerated KWin/Plasma Wayland.

## Ready-To-Flash Image

`radxa-a733-debian12-kde-pvr-20260716.img.xz` packages the verified Debian 12
KDE system, display kernel, PowerVR stack, and a genuine vendor-kernel recovery
path in one image. Flash the compressed image directly with Raspberry Pi
Imager, Balena Etcher, or another XZ-capable imaging tool.

The image defaults to `l0` (`5.15.147-21.1-a733`, PowerVR enabled). Entry `l1`
uses the independent vendor `5.15.147-21-a733` kernel, initramfs, and DT files,
and adds `module_blacklist=pvrsrvkm` for recovery. The root filesystem UUID and
GPT layout from the verified base image are preserved.

The exported image has been sanitized for first boot: NetworkManager profiles,
SSH host keys, machine identity, logs, caches, and the source board's home
directory are not included. The original config partition recreates the
default `radxa`/`rock` accounts and home directories on first boot. Its ext4
filesystem passes an offline forced `e2fsck`. The assembled artifact has not
yet been reflashed to a separate SD card; hardware results in this release
refer to the source system from which it was captured and to explicit `l0`/`l1`
tests on that board.

For desktop stability, the image sets the system Qt Quick scene graph backend
to `software`. KWin/Plasma Wayland composition remains PowerVR-accelerated.

## Build

```bash
tools/download_a733_gpu_vendor.sh artifacts/gpu/vendor
tools/build_a733_gpu_module.sh \
  artifacts/gpu/vendor/img-bxm-dkms_0.1.0-3_all.deb \
  sources/linux-a733-build/src \
  artifacts/gpu/pvrsrvkm.ko
tools/package_a733_gpu.sh \
  artifacts/gpu/pvrsrvkm.ko \
  artifacts/gpu/vendor/xserver-xorg-img-bxm_1.21.1-2_arm64.deb \
  artifacts/gpu/a733-pvr-gpu_24.2.6603887+gpu4_arm64.deb
```

## Install And Activate

The deployer selects recovery entry `l1` before installation. The GPU loads
after the root filesystem is mounted and is deliberately excluded from the
size-constrained initramfs. The deployer returns to custom-kernel entry `l0`
only after all checks pass with `--activate`.

```bash
sudo tools/deploy_a733_gpu.sh \
  artifacts/gpu/a733-pvr-gpu_24.2.6603887+gpu4_arm64.deb \
  --activate
sudo reboot
```

After reboot:

```bash
uname -r
sudo a733-pvr-control status
sudo journalctl -b -k --no-pager | grep -Ei 'pvr|rgx|gpu|firmware'
ls -l /dev/dri
a733-pvr-run vulkaninfo --summary
a733-pvr-run clinfo
a733-pvr-run eglinfo
glxinfo -B
```

The minimum pass condition is a loaded `pvrsrvkm`, clean firmware startup, a
render node, and at least one Vulkan, OpenCL, or EGL/GLES probe reporting the
PowerVR BXM rather than llvmpipe.

## Known Limits

- PowerVR library paths are still exported globally by `gpu4`; isolating them
  to KWin and explicit `a733-pvr-run` commands is the next hardening task.
- XWayland glamor can fall back to software rendering.
- Qt Quick applications use software scene graph rendering in this release;
  forcing `SceneGraphBackend=opengl` can break KScreenLocker.
- Browser GPU acceleration, HDMI audio, Bluetooth audio, and long-duration GPU
  stress tests remain to be completed.

Use `sudo a733-pvr-control disable` followed by a reboot to disable activation,
or `sudo apt remove a733-pvr-gpu` to remove the package. If graphical boot
fails, set `default l1` in `/boot/extlinux/extlinux.conf` from another Linux
system.
