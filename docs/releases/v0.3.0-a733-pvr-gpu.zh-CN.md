# A733 PowerVR GPU 第一版移植

本版本把厂商 PowerVR BXM DDK 移植到 A7z 已验证的
`5.15.147-21.1-a733` 显示内核：

- `pvrsrvkm.ko`：DDK `24.2@6603887`，BVNC `36.56.104.183`；
- 固件：`rgx.fw.36.56.104.183`；
- 匹配的 EGL/GLES/Vulkan/OpenCL/DRI 用户态；
- 独立安装于 `/opt/a733-pvr/24.2.6603887`；
- 不安装厂商 `Xorg`、`modesetting_drv.so`、`libglamoregl.so` 或 LightDM 配置。

软件包把现有系统 Xorg 的 modesetting 驱动固定到显示设备 `/dev/dri/card0`；
新增的 PowerVR `/dev/dri/card1` 和 `renderD128` 只负责加速。

Debian 为 `/usr/bin/kwin_wayland` 设置了 `cap_sys_resource`，动态链接器会因此
忽略 `LD_LIBRARY_PATH`。激活时会在 `/usr/local/lib/a733-pvr` 生成无 capability
的同版本副本，并把 KWin 的 DRM 显示后端固定到 `card0`；禁用时会删除该副本。

模块离线验证结果：`vermagic` 为 `5.15.147-21.1-a733`，DDK/BVNC
标识正确，285 个外部符号均存在于当前内核的 `Module.symvers`。A7z 实机也已
通过模块和固件启动、render node、Vulkan、OpenCL、EGL/GBM 以及加速的
KWin/Plasma Wayland 验证。

## 构建

```bash
tools/download_a733_gpu_vendor.sh artifacts/gpu/vendor
tools/build_a733_gpu_module.sh \
  artifacts/gpu/vendor/img-bxm-dkms_0.1.0-3_all.deb \
  sources/linux-a733-build/src \
  artifacts/gpu/pvrsrvkm.ko
tools/package_a733_gpu.sh \
  artifacts/gpu/pvrsrvkm.ko \
  artifacts/gpu/vendor/xserver-xorg-img-bxm_1.21.1-2_arm64.deb \
  artifacts/gpu/a733-pvr-gpu_24.2.6603887+gpu4_arm64.deb
```

## 安装和激活

部署脚本先把 extlinux 默认项切到恢复内核 `l1`。GPU 在根文件系统挂载后加载，
不会写入空间已接近上限的 initramfs。只有安装和校验完成且指定 `--activate`
后，才切回自定义内核 `l0`。

```bash
sudo tools/deploy_a733_gpu.sh \
  artifacts/gpu/a733-pvr-gpu_24.2.6603887+gpu4_arm64.deb \
  --activate
sudo reboot
```

重启后验证：

```bash
uname -r
sudo a733-pvr-control status
sudo journalctl -b -k --no-pager | grep -Ei 'pvr|rgx|gpu|firmware'
ls -l /dev/dri
a733-pvr-run vulkaninfo --summary
a733-pvr-run clinfo
a733-pvr-run eglinfo
glxinfo -B
```

最低通过条件是 `pvrsrvkm` 成功加载、固件无错误、出现 render node，并且
Vulkan、OpenCL 或 EGL/GLES 至少一种 API 报告 PowerVR BXM，而不是 llvmpipe。

## 禁用和恢复

```bash
sudo a733-pvr-control disable
sudo reboot
```

完整卸载使用 `sudo apt remove a733-pvr-gpu`。如果图形启动失败，从另一台
Linux 机器挂载根分区，并把 `/boot/extlinux/extlinux.conf` 改为 `default l1`。
