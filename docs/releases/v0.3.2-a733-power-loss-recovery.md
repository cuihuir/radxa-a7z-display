# A733 Power-loss Recovery Full Image

`v0.3.2-a733-power-loss-recovery` is the new ready-to-flash Debian 12 KDE +
PowerVR image for Radxa Cubie A7Z/A733. It rolls the `display3`/`gpu8` hotplug
update, packaged XWayland/KWin acceleration, serial diagnostics, and a tested
pre-mount ext4 recovery path into one release.

## Recovery Design

The image keeps three independent paths:

- `l0`: normal `5.15.147-21.1-a733` desktop with PowerVR acceleration;
- `l1`: vendor-kernel fallback with `pvrsrvkm` blacklisted;
- `a7z-recovery`: verbose 14.7 MB initramfs that runs `fsck.ext4 -a` before
  mounting `/dev/mmcblk0p3`.

The default remains `l0`. The U-Boot menu is reliably visible and selectable
through UART0 at `115200 8N1`; HDMI must not be relied on for menu recovery.

## Hardware Validation

Validation on July 21-22, 2026 used a separate 58.2 GiB SD card:

1. the cloned card completed a normal `l0` boot with Wi-Fi and Plasma;
2. an offline `debugfs` operation introduced an invalid ext4 group-descriptor
   checksum and free-block count;
3. normal `l0` failed to mount root and entered initramfs;
4. UART selected `a7z-recovery`;
5. early `fsck.ext4 -a` reported both errors as `FIXED` and returned status `1`;
6. the system continued to the login prompt, restored Wi-Fi, and reported the
   root filesystem state as `clean`.

The normal initramfs also excludes unused AMD/NVIDIA PCI GPU modules and
firmware. This reduces it from the unstable 42.25 MB boundary to approximately
19.8 MB without changing the A733 display, PowerVR, storage, or Wi-Fi paths.
Two consecutive normal reboots passed after this change.

The exported image contains no NetworkManager profiles, machine identity, SSH
host keys, journals, or user history. On first boot systemd creates a new
machine ID, and `a7z-ssh-host-keys.service` generates unique SSH host keys
before `ssh.service`. The final compact 12 GiB image was written back to the
spare card and reached the serial login prompt with `radxa/radxa`.

## Flash

```bash
xz -d radxa-a733-debian12-kde-pvr-recovery-20260722.img.xz
sudo dd if=radxa-a733-debian12-kde-pvr-recovery-20260722.img \
  of=/dev/<target-disk> bs=4M status=progress conv=fsync
sync
```

The target card must be at least as large as the uncompressed image. Confirm the
target device carefully because flashing destroys its existing contents.

## Known Limits

- Recovery menu selection currently requires UART when normal root mounting
  fails; HDMI does not reliably show the U-Boot text menu.
- `e2fsck -a` only repairs errors considered safe for automatic repair.
  More serious corruption still requires offline `e2fsck -fy` and a backup.
- Desktop GLX remains software-rendered; XWayland EGL/GLES and KWin use the
  packaged PowerVR paths.
